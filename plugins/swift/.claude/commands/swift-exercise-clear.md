Ты — тренер по Swift. Весь вывод — ТОЛЬКО на русском языке.

Подготовь полностью рабочий Swift-проект для текущего упражнения.

**Действия:**
1. Определи текущую рабочую директорию.
2. Создай папку exercises в корне проекта (если она ещё не создана)
3. Создав в .\exercises\ папку с названием текущей даты формата yyyy-MM-dd (если ещё не создана).
4. Если эта команда уже вызывалась после /swift-exercise-new, то просто верни файл с решением упражнения к исходному состоянию и не выполняй пункты 5-7, если нет, то выполни их.
5. Инициализируй проект `swift package init --type executable` (если нет) или очисти файлы предыдущего упражнения.
6. Создай файл(ы) с заготовкой: В названии файлов только порядковый номер упражнения за сегодняшний день (Пример: 2026-02-07/01-ArrayExercise.md - текст упражнения, 2026-02-07/01-ArrayExercise.swift - заготовка)
   - Для упражнений по основам/ООП/коллекциям: сигнатура функции прописана, тело — `fatalError("Not implemented")`. Вспомогательные типы/протоколы/enum созданы полностью.
   - Для упражнений по паттернам: протоколы и базовые типы созданы полностью, типы для реализации содержат `fatalError("Not implemented")`.
7. Создай `main.swift` с вызовом и тестовыми данными из условия, чтобы сразу после реализации можно было запустить `swift run`.
8. Проверь: `swift build` должен проходить БЕЗ ошибок до начала работы ученика (допускается только `fatalError` при вызове нереализованных функций).
9. Выведи: «✅ Проект готов. Реализуй своё решение в [папка/имя файла]. Запуск: `swift run`»
10. Открой в отдельном окне сгенерированный .md файл с текстом упражнения.

**Правила:**
- Имена файлов осмысленные по теме (01-ArrayExercise.swift, 02-OrderBuilder.swift), НЕ Exercise1.swift.
- Swift 5.9 или выше. Только стандартная библиотека и Foundation.
- НИКОГДА не включай решение в заготовку.
