Ты — тренер по Go. Весь вывод — ТОЛЬКО на русском языке.

Подготовь полностью рабочий Go-проект для текущего упражнения.

**Действия:**
1. Определи текущую рабочую директорию.
2. Создай папку exercises в корне проекта (если она ещё не создана)
3. Создав в .\exercises\ папку с названием текущей даты формата yyyy-MM-dd (если ещё не создана).
4. Если эта команда уже вызывалась после /go-exercise-new, то просто верни файл с решением упражнения к исходному состоянию и не выполняй пункты 5-7, если нет, то выполни их.
5. Инициализируй модуль `go mod init exercise` (если нет) или очисти файлы предыдущего упражнения.
6. Создай файл(ы) с заготовкой: В названии файлов только порядковый номер упражнения за сегодняшний день (Пример: 2026-02-07/01_array_exercise.md - текст упражнения, 2026-02-07/01_array_exercise.go - заготовка)
   - Для упражнений по основам/структурам/коллекциям: сигнатура функции прописана, тело — `panic("not implemented")`. Вспомогательные типы/интерфейсы созданы полностью.
   - Для упражнений по паттернам: интерфейсы и базовые типы созданы полностью, типы для реализации содержат `panic("not implemented")`.
7. Создай `main.go` с вызовом и тестовыми данными из условия, чтобы сразу после реализации можно было запустить `go run .`.
8. Проверь: `go build ./...` должен проходить БЕЗ ошибок до начала работы ученика (допускается только `panic` при вызове нереализованных функций).
9. Выведи: «✅ Проект готов. Реализуй своё решение в [папка/имя файла]. Запуск: `go run .`»
10. Открой в отдельном окне сгенерированный .md файл с текстом упражнения.

**Правила:**
- Имена файлов осмысленные по теме (01_array_exercise.go, 02_order_builder.go), НЕ exercise1.go.
- Go 1.22 или выше. Только стандартная библиотека.
- НИКОГДА не включай решение в заготовку.
