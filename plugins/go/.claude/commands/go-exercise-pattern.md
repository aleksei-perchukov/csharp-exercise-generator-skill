**Шаг 2:** Покажи меню:

**Раздел S — Принципы SOLID (адаптированные для Go):**
1. SRP — Single Responsibility Principle
2. OCP — Open/Closed Principle (через интерфейсы)
3. LSP — Liskov Substitution Principle (через интерфейсы)
4. ISP — Interface Segregation Principle (маленькие интерфейсы)
5. DIP — Dependency Inversion Principle (через интерфейсы)

**Раздел P1 — Порождающие паттерны:**
1. Singleton (sync.Once)
2. Factory Function (функции-конструкторы New...)
3. Abstract Factory (через интерфейсы)
4. Builder (Строитель)
5. Functional Options (идиоматическая альтернатива Builder)

**Раздел P2 — Структурные паттерны:**
1. Adapter (Адаптер)
2. Decorator (Декоратор / Middleware)
3. Facade (Фасад)
4. Proxy (Заместитель)
5. Composite (Компоновщик)
6. Embedding (композиция вместо наследования)

**Раздел P3 — Поведенческие паттерны:**
1. Strategy (Стратегия — через интерфейсы и function values)
2. Observer (Наблюдатель — через каналы и callbacks)
3. Command (Команда)
4. Template Method (через embedding + переопределение)
5. Iterator (range-over-func, custom iterators)
6. State (Состояние)
7. Chain of Responsibility (Middleware chaining)

**Раздел P4 — Go-специфичные паттерны:**
1. Fan-in / Fan-out
2. Pipeline
3. Worker Pool
4. Semaphore (через buffered channels)
5. Context propagation

**Правила сложности:**
- 1–3: реализовать паттерн/принцип на простом примере из реальной жизни (логгер, калькулятор, магазин).

- 7–8: комбинация двух паттернов или паттерн + SOLID.
- 9–10: проектирование мини-системы (3–5 типов), самостоятельный выбор паттернов.
